## What I learned

- I learned that public subnets are used to allow AWS resources to conect to the internet via an Internet Gateway while private subnets do not allow connection to the internet. Subnets also control traffic by verifying packets that exceed a set threshold through **network ACLs**. Network ACLs check whether the packet is on the approved packet list and allow the traffic to enter the AWS network.
- **Security Groups** also verify packets received and are *stateful* - the remeber packets that have been reviewed. By default, security groups allow all traffic to pass through however custom security groups would have to have rules set that allow or deny entry. After allowing or denying traffic into the network, security groups do not verify packets or traffic leaving the network.
- Security groups are the VPC component that checks packet permissions for an EC instance.
- **Network ACLs** on the other hand are *stateless* - they do not retain any memory of packets that have already been verified. By default, all traffic is checked when incoming and checked again when outgoing. However, their rules can also be modified.
| Feature | Security Groups | Network ACLs |
| --- | --- | --- |
| Scope | Instance-level, attached to EC2 instances | Subnet-level, associated with subnets |
| State | Stateful | Stateless |
| Rule types | Allow rules only | Allow and deny rules |
| Return traffic | Automatically allowed if inbound traffic is allowed | Must be explicitly allowed in both directions |
| Uses | Fine-grained control of traffic for individual EC2 instances | Broad control of traffic in and out of subnets |

- I also learned that as part of AWS Shared Responsibility, network ACLs, subnet configuration, security group configuration and mangement are the domain of the user and not AWS.
- The steps in creating a network resource can be summed up in three steps
    - create a VPC
    - create subnets - good rule of practice is to create two prvate subnets and two public subnets to allow for high availability
    - connect public subnets to internet gateway after creating a route table to route traffic
- AWS also offers something called the **AWS Global Networking Infrastructure** that provides a dedicated internet connectivity for client network resources. This consists of the following:
    - *AWS Route 53* which is a domain name service that converts or translates a domain name into an IP address.
        - Additionally it can route traffic to different networks using different routing policies.
        - These policies are: *Latency-based routing*, *Geolocation DNS*, and *Weighted round robin*.
        - It can also be used to register domain names.
